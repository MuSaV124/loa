# Lost Ark Node Recommend Calculator v5.4.4

## v5.4.4
- 보석 전체 시세 조회를 레벨/종류별 병렬 처리로 바꾸고 짧은 서버 캐시를 추가해 반복 조회 속도를 개선했습니다.
- 유각 전체 시세 조회에 짧은 서버 캐시를 추가하고 페이지 요청을 병렬 처리하도록 개선했습니다.
- 전설 아바타 부위 검색 fallback을 순차 전체 탐색에서 신뢰도 높은 후보 배치 단위 병렬 탐색으로 바꿔 첫 조회 대기 시간을 줄였습니다.
- 프로젝트 내부 버전 표기를 v5.4.4로 통일했습니다.

## v5.4.3
- 공식 `/auctions/options` 응답 기준으로 악세 `EtcOptions` 값을 퍼센트 표시값이 아니라 `EtcValues.Value` 코드로 보내도록 수정했습니다. 예: `2.00%`는 `2`가 아니라 `200`.
- 귀걸이/반지 연마 효과 fallback 코드도 공식 옵션 구조에 맞춰 추가했습니다.
- 악세 디버그 REQUEST payload에 검색 플랜 타입, 옵션 소스, 인코딩된 옵션 값을 함께 표시해 다음 테스트에서 원인을 더 쉽게 분류할 수 있게 했습니다.
- 프로젝트 내부 버전 표기를 v5.4.3으로 통일했습니다.

## v5.4.2
- 악세 검색에서 공식 `/auctions/options` 옵션 코드를 우선 사용하고, 실패 시에만 기존 fallback 코드를 사용하도록 변경했습니다.
- 정확 옵션 검색이 0건일 때도 최종 필터를 유지한 채 3연마 후보를 보정 검색하도록 했습니다.
- 악세 검색 서버 스캔 시간, Lost Ark API 개별 요청 시간, 프론트 시세 조회 대기 시간을 늘려 타임아웃 가능성을 줄였습니다.
- 프로젝트 내부 버전 표기를 v5.4.2로 통일했습니다.

## v5.4.1
- 악세 검색을 `고대 / 티어 4 / 3연마 / 목표 옵션 2개 / 즉시구매가 오름차순` 단일 전략으로 정리했습니다.
- 실제 `ACCESSORY_UPGRADE` 옵션이 정확히 3개인 매물만 통과합니다.
- 목표 옵션 2개는 연마 슬롯 순서와 관계없이 판정하며, 세 번째 옵션은 제한하지 않습니다.
- 힘/민첩/지능과 품질은 악세 검색 및 필터 조건에서 완전히 제외했습니다.
- 공식 `Options`가 존재하면 그것만 사용하고, 없을 때만 Tooltip 텍스트를 보조 파싱해 중복 연마 판정을 방지했습니다.
- 첫 일치 페이지와 다음 페이지만 확인하며, 불필요한 내림차순/전체 3연마 보정 스캔을 제거했습니다.


- 악세 시세 검색을 실시간 단일 탐색에서 캐시형 후보 인덱스 방식으로 변경했습니다.
- 공식 경매장 API `EtcOptions`는 후보군 축소용으로만 사용하고, 최종 통과 여부는 응답 `Options`의 `ACCESSORY_UPGRADE` 옵션명과 실제 `Value`로만 판정합니다.
- 상상/상중/리버스 상중 모두 가격 오름차순으로 스캔하며, 첫 일치 매물이 나온 페이지와 바로 다음 페이지까지 확인한 뒤 종료합니다.
- 검색 결과는 서버 메모리에 짧게 캐시하여 같은 조건을 반복 조회할 때 API 요청량과 타임아웃 가능성을 줄였습니다.
- 최종 필터에서 연마 단계, 힘/민첩/지능 기본 스탯, 품질 조건은 제외했습니다.
- 악세 디버그에 캐시 여부, 인덱스 매칭 수, 스캔 플랜, 첫 매칭 페이지 정보를 표시합니다.
- 시세 조회 프론트 타임아웃과 Vercel API 함수 최대 실행 시간을 늘렸습니다.
- 이전 v5.4.0에서 도입한 캐시형 후보 인덱스 구조를 정리했습니다.

## 배포

1. ZIP을 풀어서 GitHub 저장소에 업로드합니다.
2. Vercel 환경변수 `LOSTARK_API_KEY`가 설정되어 있는지 확인합니다.
3. Vercel에서 재배포합니다.
